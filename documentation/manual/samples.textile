h1. Sample applications

This Play 1.12 fork no longer ships in-tree sample applications. The historical @samples-and-tests/@ directory was removed during the modernization phase — see the "1.12.x release notes":releases/release1.12.x/releasenotes-1.12.x for context.

Instead, generate a fresh app with:

bc. play new myapp
play run myapp

For a frontend-integrated starter:

bc. play new myapp --frontend nuxt

The @play new@ command sets up a complete app skeleton with sensible defaults: an HTTPS-ready Netty server on port 9000, a @conf/routes@ file with one example route, a stub controller, and a @conf/application.conf@ pre-populated with documented configuration keys. Add features (CORS, OpenAPI generation, metrics) by enabling the relevant @*.enabled@ keys in your config — see the corresponding manual sections.

For framework integration tests, see @framework/test-src/integration/testapp/@ — that's the test fixture @ant integration-test@ runs against.
